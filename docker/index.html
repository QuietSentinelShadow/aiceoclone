2 Architecture

The stack uses a two-container design. NullClaw handles AI interactions through its gateway API, while the content pipeline renders final assets from the generated material.

💻
Host Machine
curl / generate-content.sh
content-net (bridge)
nullclaw-content
NullClaw Gateway
Port 3000
Memory: 256 MB
CPU: 0.5 cores
/health • /v1/chat/completions
content-pipeline
Content Pipeline
Profile: render
Memory: 2 GB
CPU: 2.0 cores
Node.js + Remotion + FFmpeg
shared-output — mounted at /output in both containers

NullClaw Gateway

Runs the 678 KB NullClaw binary in gateway mode. Exposes an OpenAI-compatible API on port 3000. Read-only filesystem with a 64 MB tmpfs scratch space. Health-checked every 30 seconds.

Content Pipeline

Node.js container with Remotion, Edge-TTS, FFmpeg, and Chromium. Only starts when the render profile is active. Waits for NullClaw to pass its health check before launching.

3 Quick Start

a

Clone and enter the docker directory

git clone https://github.com/QuietSentinelShadow/aiceoclone.git
cd aiceoclone/docker
b

Create your environment file

cp .env.example .env

Open .env and set your API key:

# .env
OPENROUTER_API_KEY=sk-or-v1-your-actual-key-here

# Or use a different provider:
# ANTHROPIC_API_KEY=sk-ant-your-key-here
# OPENAI_API_KEY=sk-your-key-here
c

Start the NullClaw gateway

docker compose up -d

This builds the NullClaw image (pulls the 678 KB binary) and starts the gateway container. The pipeline container does not start yet because it is behind the render profile.

d

Verify the health endpoint

curl http://localhost:3000/health

Expected response:

{"status":"ok","version":"2026.3.3","uptime":12}

4 Configuration

NullClaw reads its configuration from configs/content-creator.json, which is copied into the container at build time.

configs/content-creator.json 
{
  "providers": {
    "default": {
      "name": "openrouter",
      "api_key_env": "OPENROUTER_API_KEY",
      "model": "google/gemini-2.5-flash"
    }
  },
  "memory": {
    "backend": "sqlite",
    "path": "/home/nullclaw/.nullclaw/memory.db",
    "search": "hybrid"
  },
  "autonomy": {
    "level": "full",
    "confirm_dangerous": true
  },
  "sandbox": {
    "backend": "auto",
    "workspace": "/output",
    "allowed_commands": ["echo", "cat", "ls", "mkdir", "cp"]
  },
  "gateway": {
    "bind": "0.0.0.0:3000",
    "pairing": false
  },
  "resources": {
    "max_memory_mb": 256,
    "max_disk_mb": 512
  }
}

providers

Configures the AI backend. The api_key_env field tells NullClaw which environment variable holds your key, so the key never appears in the config file.

memory

SQLite-backed conversation memory with hybrid (keyword + semantic) search. The database lives on the nullclaw-data volume so it persists across container restarts.

autonomy

Set to full so NullClaw can generate content without interactive confirmation. The confirm_dangerous flag still gates destructive operations.

sandbox

Auto-selects the best sandbox backend (Landlock on Linux, Bubblewrap fallback). Only the five listed commands are allowed. All file output is scoped to /output.

gateway

Binds the HTTP API to all interfaces on port 3000. Pairing is disabled for container use (the compose network provides isolation).

resources

NullClaw-level resource caps (separate from Docker's own deploy.resources.limits). Prevents runaway allocations inside the sandbox.

5 Content Generation

The scripts/generate-content.sh script orchestrates AI content creation through the NullClaw gateway.

Run the generator 
cd docker
bash scripts/generate-content.sh

What happens step by step

1

Wait for gateway

Polls /health every second for up to 30 seconds until NullClaw responds.

2

Generate blog content

Sends configs/blog-prompt.md to the chat completions API. Saves the result to output/blog/generated-article.md.

3

Generate video script

Sends configs/video-script-prompt.md to the API. Saves the narration JSON to output/scripts/narration.json.

4

Done

Output directory structure is ready for the pipeline renderer.

Output directory structure

output/
  blog/
    generated-article.md
  scripts/
    narration.json
  audio/         # populated by pipeline
  video/         # populated by pipeline

6 Video Rendering

Once content is generated, activate the render profile to launch the pipeline container. It reads from the shared volume, synthesizes audio with Edge-TTS, and renders video with Remotion + FFmpeg.

Start the renderer 
docker compose --profile render up pipeline
🎤

Edge-TTS

Converts narration JSON to MP3 audio files using Microsoft Edge neural voices.

🎬

Remotion

React-based video framework renders animated scenes from the script data in headless Chromium.

🎥

FFmpeg

Composites audio and video tracks into the final MP4, handling encoding and muxing.

Resource note

The pipeline container is allocated 2 GB RAM and 2 CPU cores. Video rendering with Chromium is memory-intensive. If your machine has limited resources, render one video at a time.

7 Troubleshooting

API key not set or invalid

Symptom: NullClaw starts but content generation returns empty or error responses.

Fix:

# Check the container's env
docker compose exec nullclaw env | grep API_KEY

# Make sure .env has the right variable name
cat .env

Container won't start or exits immediately

Symptom: docker compose ps shows the container as exited or restarting.

Fix:

# Check logs for the crash reason
docker compose logs nullclaw --tail 50

# Rebuild if the image is stale
docker compose build --no-cache nullclaw
docker compose up -d

Out of memory during rendering

Symptom: The pipeline container is killed by the OOM killer, or Chromium crashes during Remotion renders.

Fix:

# Increase the pipeline memory limit in docker-compose.yml
deploy:
  resources:
    limits:
      memory: 4G  # was 2G

# Or reduce concurrency in Remotion by setting env var
environment:
  - REMOTION_CONCURRENCY=1

Port 3000 already in use

Symptom: Docker reports a port binding conflict on startup.

Fix:

# Change the host port in docker-compose.yml
ports:
  - "3001:3000"  # map to a free host port

# Update the gateway URL for the generator script
NULLCLAW_URL=http://localhost:3001 bash scripts/generate-content.sh

Health check keeps failing

Symptom: Container shows (unhealthy) and the pipeline refuses to start.

Fix:

# Check what NullClaw is actually listening on
docker compose exec nullclaw wget -qO- http://127.0.0.1:3000/health

# Ensure the gateway bind address matches the healthcheck
# In content-creator.json: "bind": "0.0.0.0:3000"
