Chapter 01

What is NullClaw?

NullClaw is a 678 KB static Zig binary that serves as a fully self-contained AI gateway, agent runtime, and automation platform. It has zero runtime dependencies — no Python, no Node.js, no Docker required. Just drop the binary and run.

Designed from the ground up for extreme efficiency, NullClaw starts in under 2 milliseconds and operates within roughly 1 MB of RAM. Despite its minimal footprint, it packs serious capability: support for 22+ AI providers, 18 communication channels, 18+ built-in tools, and a test suite spanning 3,230+ tests.

Whether you are running it on a $5 Raspberry Pi Zero, an edge IoT node, a CI/CD pipeline, or a full production server, NullClaw delivers the same experience: instant startup, predictable resource usage, and zero configuration drift.

Chapter 02

Quick Setup Guide

Get NullClaw running in under a minute. No dependencies to install, no containers to pull.

1

Install

Via Homebrew (macOS / Linux):

terminal 
$ brew install nullclaw

Or build from source:

terminal 
$ git clone https://github.com/nullclaw/nullclaw.git
$ cd nullclaw
$ zig build -Doptimize=ReleaseSmall
2

Onboard

Interactive wizard (recommended for first-timers):

terminal 
$ nullclaw onboard --interactive

Or one-liner for CI/scripts:

terminal 
$ nullclaw onboard --api-key sk-... --provider openrouter
3

Start Gateway

Launch the API gateway (defaults to 127.0.0.1:3000):

terminal 
$ nullclaw gateway
# Gateway listening on 127.0.0.1:3000
# OpenAI-compatible API ready
4

Connect a Channel

Spin up a Telegram bot (or any of 18 channels):

terminal 
$ nullclaw channel start telegram
5

Daemon Mode

Run NullClaw in the background as a daemon:

terminal 
$ nullclaw daemon
6

OS-Managed Service

Install as a system service (systemd / launchd / Windows Service):

terminal 
$ nullclaw service install
# Installs and starts NullClaw as an OS-managed service
# Auto-restarts on crash, starts on boot

NullClaw — The Minimalist Powerhouse

NullClaw occupies a unique position in the AI agent landscape: maximum capability in minimum space. Written in Zig with zero allocator overhead, it produces a single static binary smaller than most JPEG images. This makes it ideal for edge deployments, IoT devices, embedded systems, and any environment where resources are constrained. Its security model is defense-in-depth, layering ChaCha20-Poly1305 encryption with OS-level sandboxing via Landlock, Firejail, Bubblewrap, or Docker isolation. The trade-off is a smaller ecosystem and a steeper learning curve for contributors unfamiliar with Zig.

ZeroClaw — The Rust Middle Ground

ZeroClaw bridges the gap between NullClaw's extreme minimalism and OpenClaw's feature richness. Built in Rust, it benefits from memory safety guarantees and a mature ecosystem of crates. At 3.4 MB and under 5 MB of RAM, it is still remarkably lean compared to traditional frameworks. Its workspace-scoped security model with localhost-only defaults provides a solid baseline for production deployments. ZeroClaw is an excellent choice for teams already invested in the Rust ecosystem who want strong performance without the constraints of Zig's younger toolchain.

OpenClaw — The Feature-Rich Ecosystem

OpenClaw takes the opposite approach: maximize developer experience and extensibility at the cost of resource efficiency. Built on TypeScript with a full Node.js runtime, it offers the richest plugin ecosystem, the most integrations, and the lowest barrier to entry for web developers. However, its ~28 MB footprint and 1 GB+ RAM usage make it impractical for constrained environments. More critically, CVE-2026-25253 exposed a remote code execution vulnerability in its DM pairing mechanism, highlighting the security risks of its more permissive architecture. OpenClaw is best suited for teams that prioritize ecosystem breadth and rapid prototyping over operational efficiency.

Chapter 04

Advanced Mode

Unlock NullClaw's full potential with daemon mode, scheduled tasks, full autonomy, and its sophisticated memory and security systems.

Daemon Mode & Cron Scheduler

Running nullclaw daemon launches NullClaw as a persistent background process with a built-in cron scheduler. This enables always-on operation: scheduled code reviews, periodic report generation, automated monitoring, and proactive task execution — all without human intervention.

Full Autonomy Configuration

For advanced users who want NullClaw to operate with complete autonomy, the configuration supports unrestricted command and path access:

config.json 
{
  "autonomy": {
    "level": "full",
    "allowed_commands": ["*"],
    "allowed_paths": ["*"]
  }
}

Warning: Full autonomy mode grants unrestricted system access. Only enable this in sandboxed environments or when you fully understand the implications. Use Landlock/Firejail/Docker isolation as a safety net.

Memory System

NullClaw's memory architecture combines SQLite for structured storage with FTS5 full-text search and cosine vector similarity. Retrieval uses a 0.7 / 0.3 blend — 70% semantic (vector) relevance and 30% keyword (FTS5) relevance — giving you the best of both retrieval paradigms without requiring an external vector database.

Security Architecture

NullClaw implements a defense-in-depth security model with multiple isolation layers:

  • ChaCha20-Poly1305 — All secrets and sensitive configuration are encrypted at rest using authenticated encryption.
  • Landlock — Linux kernel-level filesystem sandboxing restricts which paths NullClaw can access.
  • Firejail — Process-level sandboxing with seccomp-bpf filters for system call restriction.
  • Bubblewrap — Unprivileged namespace sandboxing for user-space isolation.
  • Docker — Full container isolation for the most restrictive deployment scenarios.

Chapter 05

Building Your AI CEO Clone

The concept of an AI CEO clone is an autonomous decision-making agent that can strategize, delegate, and execute — operating as a tireless digital executive. Here are three approaches, from conceptual to fully realized.

01

Conceptual: What Is an AI CEO Clone?

The idea

An AI CEO clone is not a chatbot that answers questions. It is an autonomous agent with persistent memory, scheduled actions, and the authority to make decisions and execute on them. Think of it as the difference between a calculator and an accountant — one responds to inputs, the other proactively manages outcomes.

The ideal AI CEO clone can: analyze business metrics on a schedule, generate and assign tasks, monitor progress, write and review code, produce reports, and adapt its strategy based on outcomes — all autonomously, 24/7, without human prompting.

02

NullClaw Advanced: Daemon + Cron + Full Autonomy

Always-on operation

With NullClaw's daemon mode, cron scheduler, and full autonomy configuration, you can build a self-operating AI agent that runs continuously in the background. The daemon maintains persistent state, the cron scheduler triggers periodic tasks (daily standups, weekly reports, hourly monitoring), and full autonomy grants unrestricted tool access.

ceo-config.json 
{
  "autonomy": {
    "level": "full",
    "allowed_commands": ["*"],
    "allowed_paths": ["*"]
  },
  "cron": {
    "daily_standup": "0 9 * * *",
    "weekly_report": "0 17 * * 5",
    "code_review": "0 */4 * * *"
  },
  "memory": {
    "persistent": true,
    "vector_weight": 0.7,
    "keyword_weight": 0.3
  }
}
